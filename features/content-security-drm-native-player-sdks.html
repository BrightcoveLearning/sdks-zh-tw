DRM解決方案

數字版權管理（DRM）通過根據許可策略對視頻數據進行加密和解鎖來保護您的視頻內容。有關Video Cloud對DRM的支持的更多信息，請參閱概述：視頻雲中的數字版權管理（DRM）。

Brightcove Native Player SDK 平台支持以下 DRM 解決方案：

平台 DRM解決方案
適用於iOS的Native Player SDK 帶FairPlay流的HLS
適用於tvOS的Native Player SDK 帶FairPlay流的HLS
適用於Android的Native Player SDK DASH與Widevine模塊化

視頻雲客戶

在以下步驟中，您將創建啟用DRM的內容。

使用 Apple FairPlay 流媒體

如果您將Widevine Classic與iOS應用程序一起使用，則可以轉換為使用Apple FairPlay Streaming，如下所示：

  1. 如果您的帳戶未啟用DRM，請與您的客戶經理聯繫以將其啟用。

    要使用 DRM 加密，您的帳戶必須啟用這些功能。有關更多信息，請聯繫您的客戶經理。

  2. 啟用帳戶後動態交付和 DRM，您的視頻在攝取時會自動打包為 DRM。由於 DRM 不會追溯應用於 Dynamic Delivery 資產，舊內容需要重新轉碼（我們的工程團隊可以幫助將 DRM 應用於預先存在的內容）。

  3. 對受 FairPlay 保護的視頻的支持已集成到適用於 iOS 的 Brightcove Player SDK 的核心框架中。有關更多詳細信息，請參閱以下文檔：

    • 從Brightcove FairPlay插件遷移
    • 使用FairPlay的基本播放器示例
     
    要在外部屏幕上顯示 FairPlay 內容，請參閱iOS 代碼片段文檔。

使用 Google Widevine 模塊

如果您將Widevine Classic與Android應用程序結合使用，則可以按照以下方式過渡到使用Google Widevine Modular：

  1. 如果您的帳戶未啟用DRM，請與您的客戶經理聯繫以將其啟用。

    要使用 DRM 加密，您的帳戶必須啟用這些功能。有關更多信息，請聯繫您的客戶經理。

  2. 啟用帳戶後動態交付和 DRM，您的視頻在攝取時會自動打包為 DRM。由於 DRM 不會追溯應用於 Dynamic Delivery 資產，舊內容需要重新轉碼（我們的工程團隊可以幫助將 DRM 應用於預先存在的內容）。

  3. 現在，您可以在您的應用程序中使用適用於Android的Brightcove Native SDK。要查看 Widevine 模塊化內容，您需要使用 ExoPlayer 視頻視圖。有關更多詳細信息，請參閱以下文檔：

    • 使用ExoPlayer保護DRM的內容
    • 使用Widevine Modular的基本播放器示例
     
    ExoPlayer 視頻視圖 ( BrightcoveExoPlayerVideoView ) 支持 Widevine Modular，無需額外插件。

Brightcove Player客戶

使用 Apple FairPlay 流媒體

如果您將Widevine Classic與iOS應用程序一起使用，則可以轉換為使用Apple FairPlay Streaming，如下所示：

  1. 對於FairPlay，您需要產生受DRM保護的內容，該內容使用帶有分段和加密視頻的HLS清單。

  2. 由於您不會使用 Brightcove FairPlay 許可證服務器，因此您需要定義一些與 FairPlay 許可證相關的屬性

    對受 FairPlay 保護的視頻的支持已集成到適用於 iOS 的 Brightcove Player SDK 的核心框架中。有關更多詳細信息，請參閱以下文檔：

    • 從Brightcove FairPlay插件遷移
    • 使用FairPlay的基本播放器示例
     
    要在外部屏幕上顯示 FairPlay 內容，請參閱iOS 代碼片段文檔。

使用 Google Widevine 模塊

如果您將Widevine Classic與Android應用程序結合使用，則可以按照以下方式過渡到使用Google Widevine Modular：

  1. 對於Widevine Modular，您需要生成受DRM保護的內容，該內容使用帶有分段和加密視頻的MPEG-DASH清單。

  2. 適用於 Android 的 Brightcove Native SDK 默認配置為使用 Brightcove 的 Widevine 許可服務器。您的視頻需要配置為使用您的 Widevine 許可服務器。這是一個例子：

      import com.brightcove.player.display.WidevineMediaDrmCallback;
  import com.brightcove.player.media.DeliveryType;
  import com.brightcove.player.model.Video;
  String performVideoUrl = "http://url.to.your.video";
  String performLicenseUrl = "https://url.to.your.licensing.server";
  Video performVideo = Video.createVideo(performVideoUrl, DeliveryType.DASH);
  performVideo.getProperties().put(WidevineMediaDrmCallback.DEFAULT_URL, performLicenseUrl);

  3. 現在，您可以在您的應用程序中使用適用於Android的Brightcove Native SDK。要查看 Widevine 模塊化內容，您需要使用 ExoPlayer 視頻視圖。有關詳細信息，請參見以下內容：

    • 使用ExoPlayer保護DRM的內容
    • 使用Widevine Modular的基本播放器示例
     
    ExoPlayer 視頻視圖 ( BrightcoveExoPlayerVideoView ) 支持 Widevine Modular，無需額外插件。
