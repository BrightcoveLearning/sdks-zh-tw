概述

適用於 tvOS 的 Native SDK 允許您為 Apple TV 創建以視頻為中心的應用程序。

  • Video Cloud 客戶可以播放 Video Cloud 庫內容
  • Brightcove Player客戶可以播放具有Internet可訪問URL的視頻

在攝取您的內容時，我們建議使用高分辨率配置文件以獲得 Apple TV 的最佳再現。有關更多詳細信息，請參閱Apple的Apple TV的視頻格式文檔。

支持的版本

積極支持

Brightcove 在以下 tvOS 版本的最新公開版本上為最新的 tvOS SDK 提供積極支持：

  • 電視操作系統 14
  • tvOS 13.4.8
  • tvOS 12.4.1

被動支持

Brightcove為以下tvOS版本提供被動支持：

  • tvOS 11.4.1

了解支持級別

支持的定義是【主要版本】 . [次要版本] . 【補丁版本】操作系統（例如：tvOS 11.4.1）。為了利用Apple的最新技術改進，我們提供了兩個級別的支持：

  • 積極的：Brightcove將測試和修復這些操作系統版本上的錯誤。
  • 被動的：Brightcove將在這些操作系統版本上進行測試，但將根據具體情況處理錯誤修復。

通常，Brightcove對80％的用戶群使用的OS版本提供主動支持，對至少5％的用戶群使用的其他版本提供被動支持。

入門

在開始使用用於tvOS的Player SDK進行開發之前，應了解一些概念：

  1. 媒體

    本機播放器 SDK 的當前版本支持通過 Playback API 播放來自您的 Video Cloud 帳戶的內容，或作為具有 Internet 可訪問 URL 的遠程資產播放。

  2. 分析工具

    借助 Native Player SDK，Video Cloud 客戶可以自動獲得 Video Cloud 中報告的可靠分析數據。

  3. 核心SDK和依賴關係

    適用於 iOS 的 Brightcove Player SDK 及其插件可以使用可可豆依賴管理器。所有 Brightcove Player SDK 組件的 podspecs 都託管在BrightcoveSpecs 存儲庫 .

    需要 CocoaPods 1.0 或更新版本。
  4. 本機SDK參考

    為了幫助您理解 SDK，您可以在 Player SDK 中找到架構的詳細分解。了解關鍵組件如何交互以處理控製本機視頻播放器的複雜性。Player SDK安裝，快速入門和體系結構

    在這裡，您還將找到該SDK的最新beta版本（如果存在）。

  5. 問題與解答

    加入我們SDK開發者論壇 Brightcove工程師和社區會在其中回答與SDK有關的問題。搜索以前回答的問題或發布新問題。作為一個公共團體，您可以通過提供您的電子郵件地址來獲得訪問權限。

[ 1] 當前版本的 IMA 插件支持 Google 的 IMA iOS SDK v3，它支持 iOS 9.0+ 版本。有關詳細信息，請參見Google的文件。

Brightcove 原生 SDK 支持虛擬地圖包含 VAST 協議的 XML 模板，包括浩瀚 3.0和有限的支持浩瀚 4.0 .有關VAST版本兼容性的更多詳細信息，請參見Google的支持和兼容性文檔適用於iOS的IMA SDK。本機 SDK 還支持IAB 開放測量 SDK .

[2] 如果使用 Swift，則必須手動添加 Google 的 IMA 庫，而不是使用 CocoaPods。這是因為 Google 的 IMA 庫不支持動態框架，而 Swift 不支持構建靜態庫。有關詳細信息，請參見手動的的部分適用於iOS的IMA插件Brightcove Player SDK文檔。即使Brightcove的IMA插件使用動態框架，但Google IMA庫僅作為靜態框架提供。
