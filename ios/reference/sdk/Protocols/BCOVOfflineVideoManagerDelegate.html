BCOVOfflineVideoManagerDelegate 協議參考

符合 NSObject
聲明於 BCOVOfflineVideoManagerDelegate.h

概述

遵守此協議以接收有關每個視頻和視頻軌道下載的信息。

–  didCreateSharedBackgroundSessionConfiguration:

該方法在後台 NSURLSessionConfiguration 對象創建之後調用，在它之前調用’ s 用於創建共享的 AVAssetDownloadURLSession 對象’ s 用於下載視頻。

- (void)didCreateSharedBackgroundSessionConfiguration:(NSURLSessionConfiguration *)後台會話配置

討論

您可以使用此配置對象來設置為 NSURLSession.h 中的 NSURLSessionConfiguration 指定的各種選項，例如酌情處理的旗幟，或HTTPMaximumConnectionsPerHost環境。

你應該 不是 設置允許蜂窩接入財產;這是在單獨的下載任務中設置的。

接收有關已下載視頻正在進行的跟踪下載的進度通知。

參數

用於識別下載視頻的離線視頻令牌。

對於此單個 AVMediaSelection，以百分比表示的下載進度。

如果您想暫停/恢復/取消任務，您應該使用BCOV離線視頻管理器執行這些操作。

–  didDownloadStaticImagesWithOfflineVideoToken:

當與視頻關聯的靜態圖像（縮略圖和海報）下載到存儲時調用此方法。

- (void)didDownloadStaticImagesWithOfflineVideoToken:(BCOVOfflineVideoToken)離線視頻令牌

參數

離線視頻令牌

用於識別下載視頻的離線視頻令牌。

討論

下載圖像的文件 URL 作為屬性存儲在BCOV視頻使用 -[videoObjectFromOfflineVideoToken:] 從離線視頻令牌創建 使用 kBCOVOfflineVideoThumbnailFilePath 和 kBCOVOfflineVideoPosterFilePath 鍵檢索屬性。

即使無法檢索到圖像，也會調用此方法，因此請務必檢查您從海報和縮略圖路徑創建的 UIImage。文件路徑將存在，但可能沒有圖像存儲在這些位置。

BCOVOfflineVideoManagerDelegate.h

–  shouldDeleteVideoPackage:

在清理擱淺的下載時調用此方法。

- (BOOL)shouldDeleteVideoPackage:(NSString *)視頻包路徑

參數

視頻包路徑

標記為擱淺的視頻包目錄的路徑。

討論

如果您在下載過程中強制退出應用程序或在應用程序暫停時重新啟動，則下載可能會擱淺。您不需要實現此方法，除非您在 Brightcove SDK 之外開始自己的視頻下載並且不’不想刪除您的視頻。當視頻包被標記為刪除時調用此方法，這意味著 Brightcove SDK 中沒有它的記錄。

如果該文件是您認識的文件，則應返回 NO。否則，返回YES會導致視頻包被刪除。如果不實現此方法，擱淺的下載將默認被刪除。

–  offlineVideoStorageDidChange

當應用程序未運行或處於後台時激活應用程序並且離線視頻存儲更改時調用此方法，例如當用戶通過設置應用程序的常規設置中的設備存儲設置刪除下載的視頻時。

- (void)offlineVideoStorageDidChange

討論

調用時，您應該更新 UI 以反映離線存儲中的更改。

每當應用程序處於活動狀態時，就會檢測對離線視頻存儲的更改。

這個方法在主線程上調用。
