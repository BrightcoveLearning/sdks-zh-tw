概述

適用於 Android 的 Brightcove Player SDK 解決了有關 Android 設備上 HLS 視頻播放的大部分問題。有關此解決方案的詳細信息，請參見使用適用於Android的Player SDK進行HLS播放文檔。

如果您需要使用Android的MediaPlayer對於HLS播放，您可能會遇到一些限制。有關這些限制的詳細信息，請參閱以下部分。

已知限制

如果您正在使用使用適用於 Android 的 Player SDK 進行 HLS 播放，本節中的限制不適用。

如果您選擇使用 Android 的，您只會遇到本節中的問題MediaPlayer用於沒有 Brightcove 解決方案的 HLS 播放。以下是限制：

  • Android 4.x
    • 由於操作系統錯誤，HLS流僅搜索段的開頭。

      因此，非HLS流或Brightcove Player SDK（Android 4.1或更高版本）僅支持插播廣告。

    • 由於操作系統錯誤，HLS流不能始終如一地報告正確的視頻時長。

      Brightcove Player SDK（Android 4.1 或更高版本）沒有此限制。

    • 由於 Google IMA SDK 中報告的錯誤，Google IMA 插播廣告缺少計時器、計數器、“了解詳情”按鈕疊加層和一些 AdEvent 事件。

    • 我們發現某些三星設備上的 Widevine 內容存在問題。Widevine 團隊建議升級到最新的 Android 版本。
  • Android 4.4.2
    • 播放將開始，幾秒鐘後暫停，然後從頭開始播放。

常問問題

如何讓HLS在Android上運行？

為了能夠在 Android 上提供自適應流，Brightcove 的 Player SDK 為 Android 4.1 和更新版本提供 HLS 播放功能。

HLSe呢？

HLSe具有與上述相同的局限性和建議。
