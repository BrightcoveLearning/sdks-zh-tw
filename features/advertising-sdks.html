概述

Brightcove Native SDK 提供了一套選項來幫助您通過內容獲利。Native SDK 及其集成使發布商能夠決定如何最好地優化播放器中的廣告，例如前貼片廣告、中貼廣告和後貼片廣告以及隨播廣告。

筆記：

  • Brightcove Native SDK 不支持 VPAID 廣告。
  • 對於Android MediaPlayer，播放HLS視頻時使用插播廣告存在一些已知的限制。使用 Brightcove 的 Native Player SDK 解決了其中的許多問題。學習更多關於使用適用於Android的Native SDK進行HLS播放。

提示點

可以使用廣告提示點或在VMAP廣告代碼URL中定義廣告URL：

  • 對於廣告提示點，每個廣告提示點都應配置一個 VAST 廣告代碼網址。
  • 如果沒有廣告提示點，您可以使用 VMAP 廣告代碼 URL 來定義何時播放廣告。
VMAP URL 是一個廣告播放列表，其中 VAST URL 定義單個廣告。有關一般信息，請參見學習指南：視頻廣告文檔。

客戶端插件

客戶端廣告是在客戶端設備上執行廣告請求和廣告插入的傳統方法。使用這種方法，主視頻會一直播放，直到命中特定的廣告提示點。此時，將向已配置的廣告服務器發出請求，該服務器以適當的廣告詳細信息作為響應。然後播放器開始播放所請求的廣告。

Brightcove集成通過處理髮出廣告請求，解釋廣告細節和顯示廣告的工作，使此過程變得簡單。

Google的互動媒體廣告（IMA）

Brightcove是第一個與之合作的在線視頻平台Google DoubleClick並發展對Google的IMA SDK跨網絡（Flash / HTML5）和移動平台（iOS / Android）。我們與Google的緊密關係使我們可以隨時了解即將推出的功能和錯誤修復。

發布商可以使用Brightcove IMA SDK來請求和顯示由Google AdSense網絡DFP廣告管理系統提供的廣告。他們還可以使用它來解析自己的 VMAP 播放列表，只要它符合 Brightcove IMA SDK 所解釋的 VMAP 標準即可。

我們使用適用於 iOS 和 Android 的 Google 的 IMA SDK，為兩個平台提供簡單易用的集成，確保您能夠及時了解任何 Google 更新。

服務器端廣告插入 (SSAI)

服務器端廣告拼接是一種向觀眾提供廣告的創新方法。提供單個連續流，將廣告嵌入到內容中。這會刪除廣告請求並減少加載時間。這也可以防止觀看者阻止廣告。Brightcove 原生播放器 SDK 提供了適當的邏輯和廣告 UI，以確保在廣告和內容之間切換的播放體驗比以往更流暢。
