了解支持級別

支持的定義是【主要版本】 . [次要版本]操作系統（例如：Android 8.1）。為了利用Google的最新技術改進，我們提供了兩個級別的支持：

  • 積極的：Brightcove將測試和修復這些操作系統版本上的錯誤。
  • 被動的：Brightcove將在這些操作系統版本上進行測試，但將根據具體情況處理錯誤修復。

通常，Brightcove對80％的用戶群使用的OS版本提供主動支持，對至少5％的用戶群使用的其他版本提供被動支持。

入門

在開始使用Android SDK進行開發之前，您應該了解一些概念：

  1. 媒體

    本機播放器 SDK 的當前版本支持通過 Playback API 播放來自您的 Video Cloud 帳戶的內容，或作為具有 Internet 可訪問 URL 的遠程資產播放。

  2. 分析

    借助 Native Player SDK，Video Cloud 客戶可以自動獲得 Video Cloud 中報告的可靠分析數據。

  3. 搖籃

    Gradle是Android SDK構建系統的基礎，並與Android Studio集成。

  4. Maven回購

    適用於 Android Maven 工件（jar 文件）的 Brightcove Native Player 現在可在 Brightcove Maven Repo 中使用 - https://repo.brightcove.com/releases

    有關將 Maven 存儲庫添加到您的 Android 項目的詳細信息，請參閱使用適用於 Android 的本機 SDK 構建應用程序文檔。
  5. Android SDK概述

    為了幫助您理解 SDK，請了解關鍵組件如何交互以處理控製本機視頻播放器的複雜性。

  6. 使用適用於Android的Native SDK構建應用

    使用Brightcove Player SDK for Android創建您的第一個應用程序，該程序可以播放您自己的後端系統中的視頻。Video Cloud 客戶還將從 Brightcove Catalog 服務中檢索和播放視頻。

  7. SDK開發者論壇

    加入我們的SDK開發人員論壇，Brightcove工程師和社區將回答與SDK有關的問題。搜索以前回答的問題或發布新問題。作為一個公共團體，您可以通過提供您的電子郵件地址來獲得訪問權限。

  8. 視頻點播培訓

    觀看一系列培訓視頻，引導您完成在 Android Studio 中構建應用程序的步驟，該應用程序可播放視頻雲庫中的視頻和播放列表。

筆記

以下注意事項是指最新版本的 Native SDK：

  • 從版本 6.10.0 開始，Brightcove 的 Android 原生 SDK 使用 Jetpack (androidx.*) 庫而不是原始支持庫。Exoplayer 2.10.0 及更高版本需要此更改。要使用 Jetpack，您的應用必須滿足以下要求：

    • 遷移到Jetpack（AndroidX）依賴項
    • Android Gradle工具3.2.0
    • Gradle 4.6版

    有關如何遷移到JetPack的更多信息，請參閱官方Android遷移指南。

  • Brightcove Native SDK for Android 支持的 Android 最低 SDK 版本現在是 19。
  • 對於ExoPlayer v2.9.0 +，您需要啟用Java 8支持。為此，請將以下內容添加到android您的應用程序的部分構建.gradle文件。有關詳細信息，請參見Android的ExoPlayer文檔。 
    android {
    compileOptions {
      targetCompatibility JavaVersion.VERSION_1_8
    }
  }

    這將覆蓋基於項目的默認 Java 版本compileSdkVersion價值。

已從適用於 Android 的本機 SDK 中刪除對 ExoPlayer 1 的支持。如果您還沒有這樣做，請遷移到ExoPlayer 2 框架 .

核心SDK安裝

Brightcove核心SDK的安裝非常簡單。有兩種方法可以做到這一點：

  • 使用 gradle 構建系統
  • 手動下載SDK

使用 gradle 構建系統

添加 Brightcove SDK 及其依賴項的最簡單方法是使用 Android Studio 中的 gradle 構建系統。

使用帶有 '+ 的動態依賴版本' Gradle 社區不鼓勵使用字符。使用動態版本控制會給您的構建過程帶來重大風險，因為較新的 API 可能會意外地與您應用的源代碼不兼容。

Brightcove建議始終使用帶有特定版本字符串的最新版本。例如，查看build.gradle和gradle.properties中的文件android-player-samples項目。

有關使用gradle的更多信息，請參見Android 添加構建依賴項文檔。

手動下載SDK

您也可以手動下載 Brightcove SDK，如下所示：

  1. 只需下載 SDK 並解壓即可。您會找到許多文件夾。
  2. 在 - 的里面sdk文件夾中，您將找到名稱為core的SDK。 android-sdk-xxxaar， 在哪裡xxx是最新的SDK版本號。
  3. 添加android-sdk-xxxaar文件以及您可能需要的所有插件文件。

使用 IMA 廣告時，最佳做法是在實施之前檢查您的廣告是否在 SDK 之外工作。以下鏈接將對此有所幫助：

隨播廣告

Android TV不支持隨播廣告和點擊型廣告。由於Android TV不支持Web瀏覽器，因此沒有指向點擊URL的目的地。

使用 ProGuard

使用混淆器允許您創建更安全的應用程序，使它們更難逆向工程。ProGuard已集成到Android構建系統中，因此您可以選擇啟用它或在沒有它的情況下進行構建。

如果在Gradle中啟用ProGuard時收到錯誤消息，請嘗試添加以下排除規則：

  -dontwarn com.google.**
  -dontwarn android.media.**
  -keep class android.media.** { *; }
  -keep class com.google.** { *; }
  -keep interface com.google.** { *; }
  -keep class com.google.ads.interactivemedia.** { *; }
  -keep interface com.google.ads.interactivemedia.** { *; }

有關Brightcove本機SDK使用的ProGuard規則的完整列表，請下載並解壓縮當前的Android本機SDK。打開sdk文件夾並查看proguard.txt 文件。

視頻暫停

Android的智能暫停當用戶將視線從設備屏幕上移開時，三星設備上的“視頻"功能將暫停視頻。如果您不希望視頻在啟用智能暫停的設備上暫停，您可以禁用此功能。
