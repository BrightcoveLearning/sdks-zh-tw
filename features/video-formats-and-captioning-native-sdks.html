概述

在選擇要使用的視頻格式和要應用的字幕時，了解您的選項對於確保您的應用程序向觀眾提供所需的視頻體驗至關重要。

適用於移動平台的 Brightcove Native Player SDK 通過確保您的內容提供卓越的字幕體驗，努力確保您的應用程序覆蓋更廣泛的受眾並符合 FCC 準則。

有關使用字幕的標準和技術的更多信息，請查看Zencoder關於隱藏式字幕的博客文章。

查看每個類別的視頻格式和字幕：

  • 漸進式下載
  • 自適應比特率流
  • 數字版權管理（DRM）

漸進式下載

使用此方案的視頻格式強制內容從頭到尾無間隙地下載，並導致將完整的內容下載到設備。這種行為意味著觀眾將無法尋找任何尚未緩衝的區域。

基於此功能，我們建議不要將插播廣告與漸進式下載視頻格式結合使用。除此之外，應用程序發布者應意識到，清除視頻緩衝區的任何操作均會導致客戶端從內容開始時開始對其進行緩衝，如果他們隨後嘗試恢復播放。

由於 608-over-708 字幕內置於 MP4 流中，因此需要重新創建視頻文件以更改字幕。如果您想本地化視頻，這可能是一個問題。也無法向 MP4 文件添加新字幕，因為沒有可用的清單。相比之下，字幕可以添加到 HLS 文件中，因為它們確實包含清單。

自適應比特率流

使用此方案的視頻格式被分解為播放器需要時下載的短片段。這允許用戶在內容中的任何位置尋找並且只需要下載他們已經跳入的片段。通常，遵循此方案的格式使用“清單"，其中包含視頻的所有詳細信息，例如片段的數量以及在何處找到它們。

適用於iOS和tvOS的Brightcove Player SDK的說明：

  • iOS設備上的大多數在線播放都需要自適應比特率流（特別是HLS）。有關詳細信息，請參見蘋果公司有關HLS的技術文件。

  • 有關將WebVTT字幕用於HLS內容的詳細信息，請參見同步WebVTT字幕文檔。

 

與 HLS 類似，MPEG-DASH 支持通過 HTTP 傳輸高質量的媒體內容。它還可以通過將內容分成一系列文件段來工作。

數字版權管理（DRM）

值得注意的第三組格式是某些DRM合作夥伴使用的專有格式。

Brightcove Native Player SDK 無法訪問或控制這些格式的屬性和功能。使用這些格式發現的問題可能需要直接解決給其所有者。

Widevine 筆記：

  • 只有在使用 Widevine 集成時才支持 Widevine 媒體。
  • 對於Android SDK，僅Android 4.4+版本支持Widevine Modular。
  • 對於使用Brightcove Player SDK for iOS構建的iOS應用，如果需要DRM，則應使用FairPlay。

 

當使用HTTP Live Streaming（HLS）時，Apple的FairPlay Streaming（FPS）技術可用於傳遞安全內容。

  • Apple的已知問題-在以下情況下，FairPlay流內容不起作用：
