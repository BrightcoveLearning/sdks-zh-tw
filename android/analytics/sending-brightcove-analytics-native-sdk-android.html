分析整合

如果您使用Brightcove播放器，catalog包含在SDK中以檢索和播放視頻，分析數據將自動收集並顯示在Video Cloud Analytics模塊中。

如果您覆蓋BrightcoveVideoView類或不使用Brightcove播放器和目錄，則需要將帳戶ID發送到Video Cloud Analytics。

自定義屬性

從適用於Android的Brightcove Mobile SDK 4.2.7版本開始，您可以從應用程序將帳戶ID，目標URI和源URI的自定義值發送到Brightcove Analytics。

您的帳戶 ID 是必需的，但在您使用 Brightcove 播放器和catalog .

如果您願意，您可以設置目標和源 URI。默認值基於應用程序包名稱。

這些屬性可以在你的 Android 中設置MainActivity類文件：

brightcoveVideoView = (SeamlessVideoView) findViewById(R.id.brightcove_video_view);

Analytics analytics = brightcoveVideoView.getAnalytics();
analytics.setAccount("your accountID");
analytics.setDestination("valid URI");
analytics.setSource("valid URI");

帳戶

使用account屬性以在 Brightcove Analytics 中設置帳戶 ID 值。這個屬性只有在沒有的情況下才有用accountId視頻元數據中的值（例如，如果您的應用使用混合視頻雲視頻，其中包含accountId屬性和非視頻雲視頻，其中不包含accountId財產）。

例如，您可以按以下方式設置accountId屬性：

brightcoveVideoView.getAnalytics().setAccount("your accountId");

當您設置帳戶屬性時，此屬性的值將用於您尚未擁有accountId財產。此屬性最有用的值是您自己的帳戶 ID。這些視頻將在分析模塊中報告為其他/第三方因為 Brightcove Analytics 目前不支持未通過 Brightcove Catalog 服務檢索的視頻的詳細報告。

對於Video Cloud視頻，設置此帳戶替換對報告沒有影響，因為該視頻的accountId元數據值保持不變。無法通過設置此帳戶值來覆蓋該值。

目的地

使用destination屬性來設置發起事件的位置的值。目標屬性的值必須採用正確的 URI 格式，類似於此示例：

bcsdk://com.mhogben.source

目標屬性的默認值是bcsdk://後跟應用名稱，後跟包名稱。

例如，在Android SDK HLS示例應用中，目標值為：

bcsdk://HLSSampleApp/com.brightcove.player.samples.hls.basic

您應用中任何實例化屏幕的屏幕brightcoveVideoView可以將此目標替代設置為不同的值，以更好地跟踪應用本身中的視頻觀看次數。

來源

使用source屬性以設置將最終用戶發送到目標 URI 的 URI。與目標值一樣，源屬性的值必須採用正確的URI格式，類似於以下示例：

bcsdk://com.mhogben.source

源屬性的默認值為空。當值為 null 時，源屬性不會發送到 Analytics 服務器。

例如，您可以如下設置source屬性：

brightcoveVideoView.getAnalytics().setSource("bcsdk://com.brightcove.HLSPlayerSampleAppSource");

筆記：有關在目標字段和源字段中使用的URI結構的更多詳細信息，請參見URI方案維基百科頁面。你可以使用這樣的工具網址編碼器將字符轉換為 UTF-8 編碼格式。

查看分析

在視頻雲分析模塊中，您可以在帳戶級別或在詳細級別（例如通過視頻）查看指標數據。要了解如何操作，請查看以下文檔：

  • 查看 Brightcove 分析
  • 績效報告參考

筆記：Brightcove Analytics在所有報告中都會將您設置的目標值和源值轉換為所有小寫的字符串值。
