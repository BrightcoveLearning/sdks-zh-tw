適用於 iOS 的 Brightcove Native Player 參考

適用於 iOS 版 Brightcove Player SDK 的 Google Cast 插件，版本 6.9.0.1697

要求

  • Xcode 11.0+
  • 弧

支持的平台

iOS 11.0 及更高版本。

安裝

Brightcove Player SDK 的 Google Cast 插件提供了一個用於安裝的靜態庫框架。

Google Cast 插件支持適用於 iOS 的 Google Cast SDK 4.5.0 版。可以找到 Google Cast 發行說明這裡 .

可可豆

您可以使用可可豆將適用於 Brightcove Player SDK 的 Google Cast 插件添加到您的項目中。您可以找到最新的 Brightcove-Player-GoogleCast podspecs 這裡 .Pod 將自動包含正確版本的 GoogleCast。需要 CocoaPods 1.0 或更新版本。

CocoaPod Podfile 示例：

```bash源碼‘https://github.com/CocoaPods/Specs'來源’https://github.com/brightcove/BrightcoveSpecs.git'

平台：ios，‘ 11.0 ’

使用_框架！

目標‘示例應用程序’做豆莢‘ Brightcove-播放器-GoogleCast ’結束```

安裝 Brightcove-Player-GoogleCast 會自動安裝依賴項 Brightcove-Player-Core。根據您的需要，您可以選擇包含動態 Brightcove-Player-Core 框架或靜態框架。默認情況下，安裝動態 Core 框架。要選擇靜態 Core 框架，您的 Podfile 將如下所示：

```bash源碼‘ https://github.com/CocoaPods/Specs ’來源‘ https://github.com/brightcove/BrightcoveSpecs.git ’

平台：ios，‘ 11.0 ’

使用_框架！

目標‘示例應用程序’做豆莢‘ Brightcove-Player-GoogleCast-static ’結束```

更新安裝時，它’刷新 BrightcoveSpecs 存儲庫的本地副本是一個好主意，以確保您在本地擁有最新的 podspec，就像更新 CococaPods 主存儲庫一樣。用pod 回購更新這樣做。

手動的

手動將適用於 Brightcove Player SDK 的 Google Cast 插件添加到您的項目中：

  1. 跟著Google Cast SDK 手動設置指導。
  2. 跟著Brightcove Player SDK 手動安裝指導。
  3. 下載適用於 Brightcove Player SDK 的 Google Cast 插件框架。
  4. 在“一般的”應用程序目標的選項卡，將 BrightcoveGoogleCast.framework 從 Google Cast Plugin for Brightcove Player SDK 下載添加到列表框架、庫、嵌入式內容 .
  5. 在“構建設置”應用程序目標的選項卡，確保“框架搜索路徑”包括框架的路徑。這應該自動完成，除非框架存儲在與您的項目不同的根目錄下。

進口

Brightcove Player SDK 的 Google Cast 插件可以通過幾種不同的方式導入到代碼中； @import BrightcoveGoogleCast; , #import <BrightcoveGoogleCast/BrightcoveGoogleCast.h>或者#import <BrightcoveGoogleCast/[特定類].h> .您可以導入谷歌廣播和Brightcove播放器SDK模塊以類似的方式。

在你開始之前

在嘗試使用此插件之前，您應該已經熟悉以下內容：

  • 註冊您的應用程序 .這將引導您創建接收者應用程序並授予您一個應用程序 ID。一開始，您可以使用演示應用程序 ID ‘ 4F8B3483 ’ .
  • 使用適用於 iOS 的 Cast 應用程序框架 (CAF) 進行開發的設置 .這將引導您為您的應用程序設置必要的權利。
  • 初始化 Cast 上下文 .這將引導您使用接收器應用程序 ID 初始化 Cast Context。
  • 添加迷你控制器 .這將引導您了解如何利用 Google Cast SDK ’ s 迷你媒體控件。

快速開始

BrightcoveGoogleCast 插件是兩者之間的橋樑Google Cast iOS SDK和適用於 iOS 的 Brightcove Player SDK .

這個片段展示了它的基本用法。

[1] self.googleCastManager = [BCOVGoogleCastManager new]; [2] id<BCOVPlaybackController>playbackController = [BCOVPlayerSDKManager.sharedManager createPlaybackController]; [3] [playbackController addSessionConsumer:self.googleCastManager];

將代碼分解為幾個步驟：

  1. 創建一個引用BCOVGoogleCastManager單身。
  2. 創建要使用的 BCOVPlaybackController 實例。
  3. 添加您的BCOVGoogleCastManager作為會話使用者引用您的 BCOVPlaybackController 實例。

Brightcove CAF 接收器

Brightcove CAF 接收器的應用程序 ID 是341387A3並分配給常數kBCOVCAFReceiverApplicationID .您可以通過檢查應用程序 ID CAF 接收器 config.json .

如果您使用的是 Brightcove CAF 接收器，您’需要像這樣初始化這些變量：

[1] BCOVReceiverAppConfig *receiverAppConfig = [BCOVReceiverAppConfig new]; receiverAppConfig.accountId = kAccountID; receiverAppConfig.policyKey = kServicePolicyKey; [2] self.googleCastManager = [[BCOVGoogleCastManager alloc] initForBrightcoveReceiverApp:receiverAppConfig];

以下屬性也可用於設置BCOVReceiverAppConfig如所須：

  • splashScreen（用於自定義啟動畫面圖像）
  • playerUrl（用於使用自定義播放器）
  • authToken（用於 PAS/EPA）
  • watermarkingToken（用於法醫制水）
  • adConfigId（用於 SSAI）
  • userId（用於分析跟踪）
  • applicationId（用於分析跟踪）

委託方法

BCOVGoogleCastManagerDelegate具有委託方法，您可以使用它在與演員相關的主要事件發生或即將發生時收到通知，例如

  • - (void)switchedToLocalPlayback:(NSTimeInterval)lastKnownStreamPosition;
  • - (void)switchedToRemotePlayback;
  • - (void)currentCastedVideoDidComplete;
  • - (void)castedVideoFailedToPlay;
  • - (void)suitableSourceNotFound;
  • - (void)willBuildMediaInformationBuilder:(GCKMediaInformationBuilder *_Nonnull);
  • - (void)willSendMediaLoadOptions:(GCKMediaLoadOptions *_Nonnull);

有關包含說明的完整列表，請參閱BCOVGoogeCastManagerDelegate 協議參考 .

要利用這些事件，請在BCOVGoogleCastManager單身。

self.googleCastManager.delegate = self;

源選擇

這BCOVGoogleCastManager將嘗試找到合適的來源來使用。它將按以下順序查找來源：

  1. HTTPS HLS v3
  2. HTTPS DASH
  3. HTTPS MP4
  4. HTTP HLS v3
  5. HTTP DASH
  6. HTTP MP4

如果這些都沒有找到，則委託方法未找到合適的來源將被調用。

您的 Video Cloud 帳戶需要設置為支持 HLS v3，確保啟用 DASH 或為每個視頻提供可用的 MP4 源。

定制

除了委託屬性之外，還有兩個屬性可以在BCOVGoogleCastManager班級。這些是：

  • GCKImage *fallbackPosterImage：當沒有可用於視頻的海報圖像時將使用的 GCKImage。
  • CGSize 海報ImageSize：要用於創建的 GCKImage 對像圖像的高度和寬度。默認為 480h x 720w。

已知問題/限制

當使用默認的、未修改的接收器（包括演示接收器）時，以下限制適用：

  • 不支持 DRM。
  • 不支持多個音軌。
  • 不支持客戶端和服務器端廣告。
  • 不支持直播和直播 DVR 流。

使用 Brightcove CAF 接收器時，以下限制適用：

  • 不支持客戶端廣告。

支持

如果您有問題、需要幫助或想提供反饋，請使用支持門戶或聯繫您的客戶經理。要接收新 SDK 軟件發布的通知，請訂閱 Brightcove Native Player SDK 谷歌集團 .
