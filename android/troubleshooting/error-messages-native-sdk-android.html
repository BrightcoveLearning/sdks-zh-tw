錯誤訊息

適用於 Android 的 Brightcove 播放器 SDK 中定義的錯誤消息可以在SDK參考文檔。

EventType錯誤

請參閱SDK參考文檔，以獲取完整的清單事件類型消息。

場地 描述
AD_ERROR 當插件在播放廣告時遇到錯誤，它會發出此事件。
CLOSED_CAPTIONING_ERROR 表示在處理隱藏字幕信息時出錯。
ERROR 指示在處理較早事件的過程中組件發生錯誤。
ODRM_LICENSE_ERROR 當由於許可證檢索過程中發生錯誤而可以獲取離線播放許可證時，將觸發此事件。事件屬性Event.VIDEO將點視頻。請檢查屬性Event.ERROR以了解失敗的實際原因。
ODRM_LICENSE_NOT_AVAILABLE 當許可證服務器未提供離線播放許可證時，將觸發此事件。
ODRM_PLAYBACK_NOT_ALLOWED 當請求下載不符合離線播放條件的視頻時，將觸發此事件。
ODRM_SOURCE_NOT_FOUND 當請求下載不包含任何適合離線播放的源的視頻時，將觸發此事件。
SOURCE_NOT_FOUND 表示當前播放器找不到源。
SOURCE_NOT_PLAYABLE 表示源不能被當前播放器播放。
VIDEO_DOWNLOAD_FAILED 當視頻下載失敗時將觸發此事件。

傳遞消息

大多數錯誤消息都是從您的應用中被調用的組件傳遞過來的。這包括從調用返回的消息Brightcove API .

錯誤監聽器

這onError()方法處理異步操作期間發生的錯誤。僅當 JSON 響應中存在消息屬性或解析 JSON 對象時出現異常時，您才會收到此回調。

錯誤事件類型

通過本機SDK傳遞的大多數錯誤都將通過EventType.ERROR具有一些屬性，包括例外。這是如何偵聽這些錯誤的示例：

eventEmitter.on(EventType.ERROR, new EventListener() {
@Override
public void processEvent(Event event) {
	Exception exception = (Exception) event.properties.get(Event.ERROR);
	String message = (String) event.properties.get(Event.ERROR_MESSAGE);
	Video video = (Video) event.properties.get(Event.VIDEO);
	Source source = (Source) event.properties.get(Event.SOURCE);
	if (event.properties.containsKey("error_code")) {
		String errorCode = (String) event.properties.get("error_code");
		Log.e(TAG, "Error code: " + errorCode);
	}
}
});

可能有錯誤事件沒有error_code鍵，所以你需要檢查它是否存在。
