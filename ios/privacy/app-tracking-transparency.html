介紹

在 iOS 14.5 中，有一項名為 App Tracking Transparency 的新隱私功能。您可以在您的應用中使用此功能來請求用戶的跟踪許可。當用戶拒絕跟踪時，該應用程序將無法使用 Apple 的廣告商 ID (IDFA) 或其他標識符（如散列的電子郵件地址）來共享有關其廣告定位活動的數據。

如果您的應用程序收集最終用戶數據並與其他公司共享以跨應用程序和網站進行跟踪，那麼您必須使用AppTrackingTransparency框架。

適用於 iOS 和 tvOS 的原生 SDK 支持 App Tracking Transparency Brightcove IMA 和 Pulse 插件 .

要求

App Tracking Transparency 需要以下要求：

平台

以下是系統要求：

  • iOS 或 tvOS 14.5 或更高版本

Brightcove 原生 SDK

建議使用最新版本的 Native SDK。

  • 適用於 iOS 或 tvOS 版本 6.8.0 或更高版本的原生 SDK
  • 有關詳細信息，請參閱SDK 發行說明

執行

下面提到的示例應用展示瞭如何顯示應用跟踪透明度警報以確定您的應用是否可以使用廣告商 ID (IDFA)。

  1. 打開並查看以下任一示例應用程序：

    • 基本 IMAP 播放器
    • 基本脈衝播放器
  2. 打開BaseViewController，並找到viewDidLoad方法。

  3. 對於 iOS 14，SDK 檢查ATTrackingManager.trackingAuthorizationStatus .如果值為ATTrackingManagerAuthorizationStatusAuthorized，則 SDK 使用 IDFA。

    if (@available(iOS 14, *))
  {
  __weak typeof(self) weakSelf = self;
  [ATTrackingManager requestTrackingAuthorizationWithCompletionHandler:^(ATTrackingManagerAuthorizationStatus status) {
      __strong typeof(weakSelf) strongSelf = weakSelf;
      dispatch_async(dispatch_get_main_queue(), ^{
          // Tracking authorization completed. Start loading ads here.
          [strongSelf videoLibrary];
          
          [strongSelf setupPlayerView];
          [strongSelf setupPlaybackController];
          [strongSelf requestVideo];
      });
  }];
  4. 對於較舊的 iOS 版本，SDK 使用值ASIdentifierManager.sharedManager.isAdvertisingTrackingEnabled以確定是否應使用 IDFA。
