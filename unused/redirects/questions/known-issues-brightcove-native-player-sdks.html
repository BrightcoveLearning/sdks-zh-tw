將軍

大字幕文件

錄製為直播事件一部分的字幕有時會生成帶有以下提示的字幕文件：

00:00:03.760 --> 00:00:03.960 這個 00:00:03.960 --> 00:00:04.160 這個節目 00:00:04.160 --> 00:00:04.400 這個節目將 04:00: --> 00:00:04.600 此節目將是 00:00:04.600 --> 00:00:04.840 此節目將加字幕

這種字幕樣式會產生字幕效果，但可能會產生意外的副作用，即生成大小高達幾兆字節的字幕文件。在播放器中選擇這些字幕後，播放器會嘗試下載整個文件進行解析和顯示（無法流式傳輸 VTT 字幕）。這可能會導致播放器的性能問題，包括OutOfMemory錯誤。

Brightcove建議您將這些標題重新格式化為更少的行：

00:00:03.760 --> 00:00:04.840 該節目將帶有字幕

第三方框架

Brightcove不測試或不支持將Brightcove Native SDK與第三方開發框架（例如Xamarin，React Native和Titanium）集成。請與您的框架提供商聯繫以獲取集成支持。您可能會遇到以下一些問題：

  • 將適用於 Android 的 Brightcove Native SDK 與 Titanium 框架一起使用時，您可能會遇到 DRM 內容播放錯誤。這是因為 Titanium 框架覆蓋了默認Java.net.ResponseCache 用自己的TiResponseCache .

使用模擬器

無論您是使用 Android Studio 模擬器還是 Xcode 模擬器來測試視頻播放，請注意模擬器並不能準確表示實際設備的性能。雖然您可以在開發過程中使用模擬器進行初始測試，但最好使用真實設備以獲得準確的結果。

適用於Android的本機SDK

360°影片

  • 如果您不使用BrightcovePlayer課堂上，您的360°視頻可以播放，但是在暫停並恢復播放時，該應用可能會崩潰。要了解有關使用此類的更多信息，請參閱了解 BrightcovePlayer 類文檔。
  • 旋轉設備後，您可能會遇到一些視頻漂移。該問題與設備陀螺儀的校準方式有關，這會影響性能。它與特定的設備品牌和型號無關，而是與設備上安裝的運動傳感器硬件相關。較新的 Android 操作系統具有更複雜的過濾算法來糾正該問題。一個簡單的解決方法是關閉設備，然後將其放置在平坦，穩定的表面上，然後再將其打開。

即時串流

  • Native Player SDK 不支持 DASH 直播流，也不支持帶有 HLS 直播流的廣告。

離線播放

  • 使用 Android 8.0 Oreo，當後台活動關閉，下載開始，應用程序置於後台，下載停止並且在應用程序重新打開時不會繼續。通知欄顯示狀態正在下載...
  • 在 Android 8.0 Oreo 上使用 Widevine 時，您可能會注意到在某些設備上獲得的租賃許可證已過期。這是 Widevine CDM 的一個已知問題。對於大多數設備，此問題已在 Android 8.1 中得到修復。不幸的是，沒有適用於 Android 8.0 的解決方法。
  • 僅在Android 5.x設備上觀察到：當通過調用OfflineLicenseManager.releaseLicense 方法，一個MediaDrmStateException 錯誤被拋出。

  • 在裡面OfflinePlaybackSampleApp，有兩個Widevine許可證有效期。

    • absoluteExpiration是許可的到期日期，可以開始播放下載的視頻
    • playDuration視頻可以播放多長時間
     

    什麼時候playDuration變得少於 60 秒，Widevine CDM 將許可證處理為過期並拋出獲取密鑰請求失敗錯誤。播放不會從此Widevine許可例外開始。

    使用 Brightcove Native SDK for Android v6.1.0+，您可以嘗試調用AbstractOfflineCatalog.requestRentalLicense(Video, Date, long, EventListener)或者AbstractOfflineCatalog.requestPurchaseLicense(Video, EventListener)取回新的許可證。筆記：設備必須在線才能獲取新許可證。

播放本地視頻

  • 如果您嘗試播放存儲在應用程序資源文件夾 (/res/raw) 中的 MP4 視頻，您將收到異常錯誤。如果您想播放本地視頻，解決方法是將您的 MP4 視頻移動到應用程序資產文件夾。

來源選擇

  • 如果您的視頻同時具有用於WebVTT字幕/字幕的HTTP和HTTPS源，則Android的Native SDK可能不會返回HTTPS版本。Brightcove已意識到這種情況，並將在以後的版本中進行修復。

字幕

  • 如果您在 Video Cloud Studio 中為視頻分配文本軌道並設置種類字段值到字幕 .解決方法是設置種類字段值到關閉字幕當您在 Studio 中分配文本軌道時。

版本 4.11.0：已知的問題

  • 在 Fire TV 上播放 IMA 廣告時，您可能會在一個廣告播放結束和下一個廣告開始之間遇到 5 到 10 秒的延遲。在 Ad Break 中播放多個廣告時，這一點尤其明顯。這可能是 Fire TV 的問題，因為這不會發生在 Nexus 播放器上。

版本 4.10.0：已知的問題

  • 隨著 Android TV 的發布，在 Asus Nexus 播放器上播放 Widevine 內容時出現了一個已知問題。內容顯示有幾個垃圾像素。此問題已通過將 Nexus 播放器更新到 Android N (7.0) 解決。所有研究都指向硬件問題。Widevine 內容在 Nvidia Shield TV（上面有 Android TV）上播放得很好。
  • 當字幕/音頻設置超出為顯示完整列表提供的空間時，該列表在 Android TV 上不可滾動。該列表可在 Android 手機和平板電腦上滾動。

適用於iOS的本機SDK

即時串流

  • Native Player SDK 不支持帶有 HLS 直播流的廣告。

離線播放

  • 使用 iOS 12，設置kBCOVOfflineVideoManagerDisplayNameKey帶有多字節字符的字段值可能會導致下載錯誤。這是因為多字節字符被編碼為更長的字符串值，而 iOS 將字符串長度限制為 255 字節。此問題已報告給 Apple。
  • 在 iOS 12 中，didProgress當您的應用程序在下載時從後台恢復到前台時，事件可能會停止觸發。您會注意到下載狀態不再更新。此問題已報告給 Apple。
  • 當連續兩次嘗試播放同一離線視頻時，用戶可能會遇到意外的網絡活動。這AVPlayer可以切換到播放視頻的在線版本。一個錯誤報告已提交給Apple。有關解決方法的步驟，請參見兩次播放相同的脫機視頻參考文檔部分。
  • 如果您的應用程序針對的是iOS 11.0-11.2，我們不建議同時下載，因為在同時進行多次下載時，暫停和恢復行為是不可靠的。該問題已在 iOS 11.3 中得到解決。
