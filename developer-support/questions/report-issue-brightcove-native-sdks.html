概述

使用Brightcove Native Player SDK開發時出現問題或意外發生了嗎？讓我們知道發生了什麼，以便您可以重回正軌。

為了加快報告過程，請在提交錯誤之前執行以下操作：

  1. 檢查這是否是已知問題
  2. 將問題隔離到Brightcove SDK

一旦確定您的問題不是已知問題並將其隔離到Brightcove SDK，就可以提交錯誤了。請按照以下步驟操作：

  1. 提供案例特定信息
  2. 包括其他有用的信息
  3. 提交您的錯誤

檢查是否存在已知問題

首先，檢查您的問題是否已在開發者論壇檔案中報告。在這裡，您還可以註冊以包含在發布列表的郵件列表中。

  • 本地SDK開發人員論壇存檔

然後，檢查您的錯誤是否是開發人員文檔中所述的已知問題：

  • 本機SDK的已知問題

隔離問題

嘗試在提供的示例應用程序之一中重現該問題。這些應用程序提供了基本實現，可以幫助您了解問題是出在 SDK 還是外部因素上。

使用以下示例應用程序之一進行測試：

  • 適用於iOS的本機SDK示例應用程序
  • 適用於Android的本機SDK示例應用程序

提供資料

要提供特定於案例的信息，請收集以下信息：

  • 您遇到的錯誤行為/預期行為
  • 設備特定的詳細信息（品牌，型號版本）
  • 使用的 SDK 版本
  • 崩潰日誌（適用於Android的logcat，控制台/設備崩潰日誌）
  • 重現步驟

附加信息

如果可以，請包括以下詳細信息：

  • 程式碼片段
  • 顯示錯誤的示例應用程序
  • 您嘗試解決的問題

提交您的錯誤

使用以下命令向支持團隊輸入您的錯誤：

  • 創建一般支持票
