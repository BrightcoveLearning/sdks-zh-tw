BCOVCatalogService(BCOVWidevineImperativeCallbacks) 分類參考

聲明於 BCOVCatalogService+BCOVWidevineAdditions.h

概述

在這些類別方法中添加的 finder 方法的功能類似於 BCOVCatalogService 中的核心 finder 方法，不同之處在於這些方法不返回傳遞結果（或錯誤）的信號，而是返回 void 但獲取一個塊，該塊被調用結果（或錯誤）。如果你習慣使用 ReactiveCocoa ’ s 信號 API，BCOVCatalogService 提供的信號方法提供了更好的靈活性。這些類別方法適用於不想直接使用信號的客戶。

–  findWidevinePlaylistWithPlaylistID:parameters:completion:

通過播放列表 ID 從媒體 API 服務中檢索 BCOVPlaylist。

- (void)findWidevinePlaylistWithPlaylistID:(NSString *)播放列表ID 參數：（NSDictionary *）參數 完成:(void (^) ( BCOVPlaylist *playlist , NSDictionary *jsonResponse , NSError *error ))完成處理程序

參數

播放列表ID

要查找的播放列表的 ID。
參數

添加到媒體 API 請求的其他 NSString 查詢參數。如果這些值發生衝突，它們將覆蓋默認值。
完成處理程序

請求完成時將調用的塊。completionHandler 的執行將發生在主線程上。

討論

通過播放列表 ID 從媒體 API 服務中檢索 BCOVPlaylist。

如果媒體 API 返回標準 JSON 格式的錯誤響應，反序列化的響應將是回調’錯誤’秒用戶信息 .

如果發生 JSON 解析錯誤，原始 NSData 響應將包含在回調中’錯誤’秒用戶信息，由 kBCOVCatalogJSONDeserializationErrorRawDataKey 鍵控。
