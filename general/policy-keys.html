介紹

策略鍵用於使用 Playback API 從您的 Video Cloud 庫中檢索視頻和播放列表內容。由於策略鍵是在播放器上設置的，它允許您在播放器級別限制對您的內容的訪問。您還可以通過使用影雲工作室媒體模塊 .

創建 Brightcove 播放器時，會自動為每個播放器生成一個策略密鑰。此密鑰與您的 Brightcove 帳戶相關聯，並控制可以查看哪些視頻以及何時可以查看，條件是視頻存儲在 Video Cloud 中。例如，需要策略密鑰以基於以下條件限制訪問：

  • 帳戶
  • 域限制（如果使用）
  • 地理位置限制（如果使用）

有關策略密鑰的更多詳細信息，請參見概述：政策API文檔。

筆記：
  • 策略密鑰僅用於控制存儲在 Video Cloud 中的視頻。如果使用獨立的Brightcove Player，則無需訪問或檢索策略密鑰。
  • 默認情況下，未為新Brightcove Player生成的策略密鑰啟用搜索。要從 Playback API 搜索視頻，您必須使用 Policy API 創建支持搜索的新策略密鑰。有關詳細信息，請參見策略API概述文檔。

獲取策略密鑰

策略密鑰會自動為您生成和管理。因此，在大多數情況下，您不需要請求或管理它。檢索策略密鑰的最簡單方法是執行以下操作：

  1. 打開選手們模塊，然後創建一個新播放器或找到您要向其添加插件的播放器。
  2. 選擇播放器的鏈接以打開播放器的屬性。

  3. 在左側導航菜單中，選擇JSON編輯器。

  4. 在顯示的JSON代碼中，您將看到策略密鑰：

在極少數情況下，您可能尚未創建Brightcove Player，但仍需要策略密鑰（例如嚴格使用Brightcove Native Player SDK開發）。在這種情況下，您需要聯繫客戶支持獲取您的策略密鑰。

使用策略密鑰

策略密鑰可用於驗證對 Playback API 的請求。Playback API 用於在運行時在網頁或移動應用程序中用於一般媒體管理和與其他系統的集成。有關更多詳細信息，請參見播放API索引頁。

策略密鑰與API憑證

策略密鑰用於訪問 Playback API，而 API 憑據用於訪問您在其他 Video Cloud API 中的內容。

政策金鑰

策略密鑰用於客戶端檢索您的 Video Cloud 庫資產。詳細信息如下：

  • 使用播放API訪問您的視頻/播放列表數據
  • 只讀
  • 您的視頻/播放列表數據由 Playback API 緩存在服務器上
  • 使用政策API執行以下操作：
    • 創建策略密鑰。您需要聯繫客戶支持以撤銷策略密鑰。
    • 在策略級別設置域限制和地理限制。限制是由使用策略密鑰的播放器強制執行的。
您還可以使用Brightcove Player 的目錄方法從您的視頻雲庫中檢索內容。

API憑證

API身份驗證憑據（也稱為客戶端憑據）允許服務器端應用程序管理您的Video Cloud庫資產。詳細信息如下：

  • 使用Video Cloud API訪問您的數據。您將需要要使用的帳戶和 API 的客戶端憑據。
  • 讀取，插入，更新，刪除
  • CMS API 不會將您的視頻/播放列表數據緩存在服務器上
  • 用Video Cloud 的 API 身份驗證 UI創建和管理 API 憑證

管理策略密鑰

您可以按如下方式管理您的策略密鑰：

取得

無法更新策略密鑰。你可以獲取策略密鑰從 Video Cloud 帳戶中的現有播放器，或者您可以使用政策API .

撤銷

要撤銷策略密鑰，您需要聯繫客戶支持 .

刪除玩家時不會撤銷策略密鑰。這意味著，如果您使用某個播放器的策略密鑰，然後刪除該播放器，您仍然可以繼續使用該策略密鑰。
