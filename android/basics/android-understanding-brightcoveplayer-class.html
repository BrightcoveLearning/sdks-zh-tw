BrightcovePlayer類

這BrightcovePlayer類駐留在適用於 Android 的核心 Brightcove SDK 中，並擴展了 Android 的活動班級。

作為基本Activity， 這BrightcovePlayer類包裝一個BrightcoveExoPlayerVideoView並設置一個EventLogger和默認MediaController。它實現了基本的生命週期管理和全屏邏輯。它應該被子類化。

public class MainActivity extends BrightcovePlayer {
...
}

至少onCreate()該方法應使用邏輯來覆蓋BrightcoveExoPlayerVideoView變量layout.xml文件。例如：

brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

有關實現的詳細信息，請參見BasicSampleApp代碼示例。

支持庫組件

如果您選擇不使用BrightcovePlayer班級BrightcovePlayerActivity和BrightcovePlayerFragment課程可供您使用。要使用這些類中的任何一個，您需要包含appcompat你的插件構建.gradle文件如下：

implementation "com.brightcove.player:android-appcompat-plugin:${anpVersion}"

BrightcovePlayerFragment類

這BrightcovePlayerFragment類擴展了android.support.v4.app.Fragment班級。

如果要將Brightcove Player與佈局中的其他設計元素結合使用，可以使用BrightcovePlayerFragment班級。這在為平板電腦或更大的屏幕設計時很有用，因為它們有更多的空間來組合 UI 組件。例如，使用平板電腦，您可以同時顯示多個片段，它們都位於同一活動中。

public class MainFragment extends BrightcovePlayerFragment {
...
}

這onCreateView()應該擴展以將片段的佈局連接到baseVideoView調用前的實例變量super.onCreateView() .

baseVideoView = (BaseVideoView) result.findViewById(R.id.brightcove_video_view);

有關實現的詳細信息，請參見AppCompatFragmentSampleApp代碼示例。

BrightcovePlayerActivity類

這BrightcovePlayerActivity類擴展了android.support.v7.app.AppCompatActivity班級。

如果您選擇使用活動，則可以使用BrightcovePlayerActivity班級。

public class MainActivity extends BrightcovePlayerActivity {
...
}

有關實現的詳細信息，請參見AppCompatActivitySampleApp代碼示例。

生命週期管理

最佳做法是從BrightcovePlayer活動或BrightcovePlayerFragment，因為它們提供了默認的生命週期管理實施，包括以下內容：

  • 當用戶暫停應用程序時，該類會跟踪他們停止的位置。
  • 當您的應用程序從後台返回時，該類會記住視頻是否正在播放，並從中斷處重新開始播放。
  • 如果您的應用由於內存問題而被終止，則該課程將嘗試在視頻中計時到原來的位置。
  • 設置了事件記錄器，該事件記錄器將所有事件記錄到logcat中。這對調試很有幫助。
  • 該類調度事件以與插件通信。
