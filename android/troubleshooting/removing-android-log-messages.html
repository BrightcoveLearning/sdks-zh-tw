介紹

使用默認日誌消息測試您的視頻應用程序以協助調試。當您準備好發布您的應用程序時，您可以刪除這些日誌消息。

刪除日誌消息

要從適用於 Android 的本機 SDK 中刪除默認日誌消息，請執行以下步驟：

  1. 在您的應用程序中，打開proguard-rules.pro文件。
     
    在 Android Studio 中，這個文件是默認創建的。如果您想知道此文件在我們的 Brightcove Android 示例中的位置，我們已將此文件重命名為proguard-project.txt .
  2. 添加以下代碼塊以從您的應用中刪除日誌： 
    -assumenosideeffects class android.util.Log {
  public static *** v(...);
  public static *** d(...);
  public static *** i(...);
  public static *** w(...);
  public static *** e(...);
}
    有關日誌消息的詳細信息，請參閱Android 日誌類文檔。
  3. 接下來，打開您應用的build.gradle文件。
  4. 在裡面buildTypes部分，添加minifyEnabled值為的屬性true。
  5. 在之前的代碼下方，添加一個proguardFiles屬性如下：
    • 使用getDefaultProguardFile()獲取方法proguard-android-optimize.txt來自 Android SDK 的文件。
    • 參考proguard-rules.pro文件。

    您的構建.gradle文件應與此類似： 
    android {
    compileSdkVersion 25
    buildToolsVersion '25.0.0'

    defaultConfig {
        ...
    }

    buildTypes {
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'),
            'proguard-rules.pro'
        }
    }
}
在使用 Instant Run 時，Android Studio 會暫時禁用 Java 代碼覆蓋庫 (JaCoCo) 和 ProGuard。由於Instant Run僅適用於調試版本，因此不會影響您的發行版本（ https://developer.android.com/studio/run/index.html）。

如果要刪除調試構建類型的日誌，則需要禁用即時運行 .
