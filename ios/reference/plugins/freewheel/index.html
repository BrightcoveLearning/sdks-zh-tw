要求

支持的平台

tvOS 11.0 及以上。

安裝

適用於 Brightcove Player SDK 的 FreeWheel 插件提供了一個用於安裝的靜態庫框架。未來 FreeWheel 發布 dylib 版本時將添加動態框架。

適用於 Brightcove Player SDK 的 FreeWheel 插件已與 FreeWheel AdManager 框架的 6.38.0 版集成，該框架可從AdManager SDK 集成下載頁。

可可豆

您可以使用可可豆將適用於 Brightcove Player SDK 的 FreeWheel 插件添加到您的項目中。你可以找到最新的Brightcove-Player-FreeWheel podspec 這裡 .要使用此規範，請將以下內容添加到 Podfile 的頂部：來源 ' https://github.com/brightcove/BrightcoveSpecs.git' .需要 CocoaPods 1.0 或更高版本。

自由輪 SDK 不是包含在這個吊艙中。你必須手動將 FreeWheel SDK AdManager.framework 添加到您的項目。但是，該 pod 將添加 AdManager.framework 框架所需的所有庫。

靜態框架示例：

來源'https://github.com/brightcove/BrightcoveSpecs.git'  pod 'Brightcove-Player-FreeWheel'

更新安裝時，它’刷新 BrightcoveSpecs 存儲庫的本地副本是一個好主意，以便您在本地擁有最新的 podspec，就像更新 CococaPods 主存儲庫一樣。

通常，如果您使用豆莢更新在終端中，這將自動發生，或者您可以使用pod repo 更新 Brightcove .（如果您明確將 BrightcoveSpecs 存儲庫添加到 podspecs 存儲庫列表中，則它可能具有不同的名稱。）

手動的

手動將適用於 Brightcove Player SDK 的 FreeWheel 插件添加到您的項目中：

  1. 安裝最新版本的Brightcove 播放器 SDK並按照說明安裝 [ 靜態框架 ][ https://github.com/brightcove/brightcove-player-sdk-ios/ #手動安裝]。
  2. 從我們的網站下載插件的最新壓縮版本發布頁面 .
  3. 添加‘ BrightcoveFW.framework ’到您的項目。您可以通過右鍵單擊 Frameworks 文件夾並選擇“添加文件到”選項並從存儲路徑中選擇 BrightcoveFW.framework。
  4. 在“構建設置”應用程序目標的選項卡，確保“框架搜索路徑”包括框架的路徑。這應該自動完成，除非框架存儲在與您的項目不同的根目錄下。
  5. 在“構建階段”應用程序目標的選項卡，將以下內容添加到“將二進製文件與庫鏈接”階段：
    • BrightcoveFW.framework
  6. 在“構建設置”應用程序目標的選項卡：
    • 確保這件事-ObjC已添加到“其他鏈接器標誌”構建設置。
  7. 安裝 FreeWheel 庫，該庫必須從您的 FreeWheel 帳戶中檢索。

進口

Brightcove Player SDK 的 FreeWheel 插件可以通過幾種不同的方式導入代碼； @import BrightcoveFW; , #import <BrightcoveFW/BrightcoveFW.h>或者#import <BrightcoveFW/[特定類].h> .

播放和暫停

Brightcove FreeWheel 插件實現了自定義播放和暫停邏輯，以確保盡可能流暢的廣告體驗。因此，您需要確保在BCOVPlaybackController或者[BCOVSessionProviderExtension fw_play]或者[BCOVSessionProviderExtension fw_pause] ( BCOVSessionProviderExtension )，而不是 AVPlayer。

例如，第一次調用 play BCOVPlaybackController允許 Brightcove FreeWheel 插件處理前貼片廣告，而無需在前貼片前播放任何內容。

使用內置 PlayerUI

如果您使用的是 Brightcove Player SDK 5.1 或更高版本，則可以利用 Brightcove FreeWheel 插件的內置廣告控件。

要使用 PlayerUI，請按照上面的快速入門代碼創建播放控制器。

BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController>playbackController = [sdkManager createFWPlaybackControllerWithAdContextPolicy:[self adContextPolicy] viewStrategy:nil];

然後創建您的播放器視圖。此視圖包含視頻播放器視圖和顯示播放和廣告控件的視圖。無論您使用什麼插件，此設置都是相同的。

筆記：這Brightcove 播放器用戶界面不再需要模塊並已刪除。（在 Brightcove Player SDK 5.1 版之前，Brightcove PlayerUI 插件是一個單獨的框架和模塊。）您可以刪除任何引用 Brightcove PlayerUI 模塊的導入。所有 PlayerUI 標頭現在都可以在Brightcove播放器SDK模塊。

在您的 UIViewController 中創建一個屬性以跟踪 BCOVPIPlayerView。BCOVPUIPlayerView 將包含播放控制器’ s 視圖和控件視圖。

// PlayerUI 的播放器視圖 @property (nonatomic) BCOVPUIPlayerView *playerView;

創建 BCOVPUIBasicControlView，然後創建 BCOVPPUIPlayerView。這是我們將播放控制器（以及它播放的所有視頻）與控件關聯的地方。

// 創建和配置控制視圖。BCOVPUIBasicControlView *controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlsView:controlView]; // 將 BCOPUIPlayerView 添加到您的視頻視圖中。 [self.videoView addSubview:self.playerView];

自定義插件行為

您可以通過創建一個實例來自定義默認插件行為BCOVFWSessionProviderOptions並覆蓋默認屬性。使用一個BCOVFWSessionProviderOptions選項實例，您需要創建BCOVFWSessionProvider使用[BCOVSDKManager createFWSessionProviderWithAdContextPolicy:upstreamSessionProvider:options:] .

BCOVFWSessionProviderOptions *options = [[BCOVFWSessionProviderOptions alloc] init]; options.cuePointProgressPolicy = [BCOVCuePointProgressPolicy progressPolicyProcessingCuePoints:BCOVProgressPolicyProcessFinalCuePoint resumingPlaybackFrom:BCOVProgressPolicyResumeFromContentPlayhead ignoringPreviouslyProcessedCuePoints:YES]; id<BCOVPlaybackSessionProvider> sessionProvider = [sdkManager createFWSessionProviderWithAdContextPolicy:[self adContextPolicy] upstreamSessionProvider:nil options:options]; id<BCOVPlaybackController>playbackController = [sdkManager createPlaybackControllerWithSessionProvider:sessionProvider viewStrategy:nil];

預加載插槽

根據網絡條件，一旦廣告位置被點擊，廣告播放所需的時間可能會有延遲。為了提高廣告性能，可以預加載 FreeWheel 插槽。要打開預加載，請啟用預載插槽在BCOVFWSessionProviderOptions .

啟用後，將盡快加載預卷插槽。對於插播廣告，插槽將預加載可配置的秒數，由廣告到預加載時間在BCOVFWSessionProviderOptions .

使用準確的持續時間

根據您的 FreeWheel 配置，在調用中提供準確的持續時間設置視頻ID：是接收到正確數量的插播廣告與否之間的差異。

默認情況下，SDK 將等待調用BCOVFWSessionProviderAdContextPolicy直到 AVPlayerItem 具有準確的持續時間，並且該值將提供給塊調用。如果您不想等到 AVPlayerItem 持續時間準備好並且您可以從其他來源（如 BCOVVideo/BCOVSource 屬性或您自己的 CMS）獲得此信息，等待AVPlayer持續時間可用於禁用等待。

處理尋道

在搜索多個廣告連播時（如兩個插播廣告位，在不同位置），SDK 提供了一種便利機制來確定播放哪些連播。這可以通過更改默認值來修改BCOVCuePointProgressPolicy在BCOVFWSessionProviderOptions .欲了解更多信息BCOVCuePointProgressPolicy，請諮詢 BCOVFWSessionProvider 。H文件。

跟踪 FreeWheel 錯誤

FreeWheel 錯誤事件 ( FWAdErrorEvent ) 通過FWAd事件通知將傳遞給播放控制器：播放會話：didReceiveLifecycleEvent：具有事件類型的委託方法kBCOVFWLifecycleEventAdError .FreeWheel SDK 隨通知一起發送的整個 userInfo 對象包含在 events 屬性中，可以使用kBCOVFWLifecycleEventPropertyKeyAdError鑰匙。

以下是如何捕獲這些錯誤的示例：

- (void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent { if ([kBCOVFWLifecycleEventAdError isEqualToString:lifecycleEvent.eventType]) { NSDictionary *errorInfo = lifecycleEvent.properties[kBCOVFWLifecycleEventPropertyKeyAdError]; NSLog(@"FreeWheel Error Encountered: %@", errorInfo[FWInfoKeyErrorInfo]); } }
