視頻播放實現

您可以通過以下幾種方式將視頻播放器添加到您的應用中：

  • 使用 Web 視圖
  • 使用本機播放器
  • 使用 Brightcove 原生 SDK

使用 Web 視圖

一種在您的應用中實施視頻播放的選項，它可以使用webView。這將創建一個像瀏覽器一樣處理您的內容的視圖，從而可以使用 HTML 和 JavaScript。一些應用程序開發人員在應用程序中使用 HTML5 視頻播放器或 Brightcove 網絡播放器webView，但這與使用適用於 iOS 和 Android 的 Brightcove Native SDK 不同。

而一個webView 簡化了實施過程，但有許多問題使這種方法不可取：

  • 這webView 對處理器造成沉重負擔，比本地播放器消耗更多的內存。
  • 播放webView 具有與在移動設備上的瀏覽器中播放相同的問題。該實現迫使您放棄對僅全屏設備上的許多 UI 和播放相關功能的控制。這意味著犧牲一些功能，例如疊加。用戶將無法點擊疊加廣告。您還將失去阻止用戶跳過廣告的能力。
  • 我們的許多合作夥伴明確指出，他們在webView 而是提供一個iOS或Android SDK來實施。

由於這些原因，Brightcove不建議使用webView，我們不會在此平台上進行積極測試。

更好的解決方案是使用Brightcove本機SDK。

使用本機播放器

iOS和Android都提供了自己的本地播放器，您可以實施這些播放器來為視頻內容提供基礎平台。但是，在這兩種情況下，這只是一個基本參與者，沒有任何結構可輕鬆支持分析，廣告或DRM等功能。

即使本地播放器有大量在線文檔，您也可能會發現軟件更新不是定期進行的。這意味著您遇到的問題可能會在一段時間內得不到解決。當更新發佈時，尤其是 Android 操作系統，市場上使用的設備碎片化可能會導致這些修復無法普遍應用。要了解有關本機設備播放器的更多信息，您可以從安卓原生播放器，以及iOS 原生播放器選項。

更好的解決方案是使用Brightcove本機SDK。

使用 Brightcove 原生 SDK

Brightcove Native SDK 建立在本機播放器框架之上。這使您可以利用原生操作系統的速度、性能和靈活性，並將其擴展為易於集成的完整視頻體驗。此外，Brightcove Native SDK遵循已建立的命名標準，內存管理和設計模式的約定，以使入門變得盡可能簡單。

Brightcove Native SDK 可在每個平台的不同版本之間無縫運行。這可確保您覆蓋絕大多數市場，並讓您確信 Brightcove SDK 可在所有受支持的版本上運行。

與Brightcove網絡播放器一樣，Brightcove本機SDK也通過與合作夥伴的集成而獲得了巨大的價值。作為視頻領域的專家，我們通過提供與廣告，分析和數字版權管理策略。這使您可以創建適合您需求的自定義視頻體驗。
