了解支持級別

支持的定義是【主要版本】 . [次要版本] . 【補丁版本】操作系統（例如：iOS 11.4.1）。為了利用Apple的最新技術改進，我們提供了兩個級別的支持：

  • 積極的：Brightcove將測試和修復這些操作系統版本上的錯誤。
  • 被動的：Brightcove將在這些操作系統版本上進行測試，但將根據具體情況處理錯誤修復。

通常，Brightcove對80％的用戶群使用的OS版本提供主動支持，對至少5％的用戶群使用的其他版本提供被動支持。

入門

在開始使用iOS版Player SDK進行開發之前，您應該了解一些概念：

  1. 媒體

    本機播放器 SDK 的當前版本支持通過 Playback API 播放來自您的 Video Cloud 帳戶的內容，或作為具有 Internet 可訪問 URL 的遠程資產播放。

  2. 分析工具

    借助 Native Player SDK，Video Cloud 客戶可以自動獲得 Video Cloud 中報告的可靠分析數據。

  3. 核心SDK和依賴關係

    適用於 iOS 的 Brightcove Player SDK 及其插件可以使用可可豆依賴管理器。所有 Brightcove Player SDK 組件的 podspecs 都託管在BrightcoveSpecs 存儲庫 .

    需要 CocoaPods 1.0 或更新版本。
  4. 本機SDK參考

    為了幫助您理解 SDK，您可以在適用於 iOS 的 Player SDK 中找到架構的詳細細分。了解關鍵組件如何交互以處理控製本機視頻播放器的複雜性。用於iOS安裝，快速入門和體系結構的Player SDK

    在這裡，您還將找到該SDK的最新beta版本（如果存在）。

  5. 本機SDK概述

    要快速瀏覽，請了解Brightcove Player SDK組件如何與本機iOS組件交互。適用於iOS的Player SDK概述

  6. 問題與解答

    加入我們SDK開發者論壇 Brightcove工程師和社區會在其中回答與SDK有關的問題。搜索以前回答的問題或發布新問題。作為一個公共團體，您可以通過提供您的電子郵件地址來獲得訪問權限。

  7. 視頻培訓

    觀看一系列培訓視頻，這些視頻將引導您完成在 Xcode 中構建 Swift 應用程序的步驟，該應用程序可播放來自 Video Cloud 庫的視頻。

合併插件

使用適用於 iOS 的 Native SDK，BCOVSessionProviders 從每個插件創建並鏈接在一起。然後使用該鏈來創建BCOVPlaybackController .有關詳細信息，請參見組合插件部分 SDK參考。

使用 IMA 插件做廣告

VAST 廣告的展示位置通常由 VMAP 文檔指定。VAST 描述廣告，而 VMAP 描述廣告在視頻中的位置。有關代碼的詳細信息，請參見BasicIMAPlayer iOS樣本。

使用 IMA 廣告時，最佳做法是在實施之前檢查您的廣告是否在 SDK 之外工作。以下鏈接將對此有所幫助：

平台說明

以下注意事項與iOS平台相關。

使用 Airplay 鏡像您的屏幕

AirPlay 功能與適用於 iOS 的本機 SDK 開箱即用，允許用戶使用 Apple TV 將視頻流式傳輸到高清顯示器。

您可以使用 Brightcove 控件，或使用您自己的 AirPlay 自定義控件。有關詳細信息，請參閱AirPlay 的自定義控件代碼片段。

在較舊的（第 3 代）設備上使用 AirPlay 可能會導致意外行為。目標設備可能不知道在發送設備上選擇了哪些文本軌道。
