繼承自 NSObject
聲明於 BCOVPlaybackService.h

概述

BCOVPlaybackService 類提供異步方法，用於從 Brightcove 的 Playback API 檢索有關視頻的信息。

  共享URL會話

NSURLSession 在對 BCOVPlaybackService 的所有網絡調用之間共享。該對像是在初始化播放服務時創建的。如果你想使用你自己的 NSURLSession 你可以在這裡設置它。如果設置為 nil，將重新創建默認的 NSURLSession。

@property (nonatomic, strong) NSURLSession *sharedURLSession

參數

帳戶ID

返回值

一個初始化的實例。

–  findPlaylistWithPlaylistID:parameters:completion:

- (void)findPlaylistWithPlaylistID:(NSString *)播放列表ID 參數：（NSDictionary *）參數 完成:(void (^) ( BCOVPlaylist *playlist , NSDictionary *jsonResponse , NSError *error ))完成處理程序

參數

播放列表ID

要查找的播放列表的 ID。
參數

添加到播放 API 請求的其他 NSString 查詢參數。如果這些值發生衝突，它們將覆蓋默認值。可以使用 kBCOVPlaybackServiceParameterKeyOffset 和 kBCOVPlaybackServiceParameterKeyLimit 參數。看“播放列表分頁”以上。
完成處理程序

請求完成時將調用的塊。completionHandler 的執行將發生在主線程上。

討論

檢索一個BCOV播放列表從 Playback API 服務通過其在後台隊列中的播放列表 ID。

如果 completionHandler 提供了一個 NSError，則BCOV播放列表將為零。
