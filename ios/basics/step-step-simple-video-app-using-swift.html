    逐步：使用 Swift 的簡單視頻應用

    在本主題中，您將學習如何使用適用於 iOS 的 Brightcove 播放器 SDK 來建置簡單的視訊播放應用程式。您將使用 Swift 編程語言構建它。

    概覽

    此範例會播放從 Brightcove 播放 API 傳回的視訊，這是最新且建議的 API，可從您的視訊雲端庫擷取內容。

    對於此選項，您將需要原則金鑰。如果您不熟悉原則金鑰，請參閱原則 API 概觀。

    這個例子是用Swift編寫的，但是您也可以嘗試使用Objective-C。請參閱以下內容：
    • 一步步：：使用Obj-C的簡單視頻應用
    • VideoCloudBasicPlayer的Obj-C版本

    開始使用

    此範例會從內部視訊檔案陣列中播放視訊。

    請依照下列步驟熟悉設定使用 iOS 版 Brightcove 播放程式 SDK 的應用程式專案。您可以檢視下列各項的完整程式碼：

    • 應用程式代表

    • 查看控制器

    有兩種方法可以嘗試這個範例：

    下載範例

    下載整個 Xcode 項目進行實驗。

    1. 將 iOS 版本的原生 SDK 複製或下載至您的本機系統。
    2. 導航到播放器/視頻雲BasicPlayer / swift示例應用程序。
    3. 執行pod install指令。
    4. 開啟並執行新建立的工作區。

    如需詳細資訊，請參閱適用於 IOS/TSOS 的原生 SDK 執行範例應用程式文件。

    依照本指南中的步驟建置應用程式

    請依照下列步驟熟悉設定使用適用於 tvOS 的 Brightcove 播放程式 SDK 的應用程式專案。您可以檢視下列各項的完整程式碼：

    2. 選擇的iOS和單視圖應用程序為您的新項目。選擇下一個。

    將 SDK 及其依賴項添加到您的項目

    將 SDK 及其依賴項添加到您的項目中的最簡單方法是使用 Cocoapods。

    Cocoapods 是一個依賴管理器，將庫添加到您的項目中。它不是必需的，但使安裝更容易。若要安裝 Coapods，請參閱 Cocoapods 網站上的說明。

    • 需要版本 1.0 或更新版本。
    • 在現有的 Xcode 項目上使用 Cocoapods（該pod工具）將修改項目文件。您可能需要在執行此操作之前進行備份。

    1. 在您的項目文件夾中，創建一個名為 Podfile （無文件擴展名）的純文本文件。

    2. 使用文本編輯器，將以下代碼行添加到 Podfile 並保存它。此程式碼會執行下列作業：

      • 第 1 行：指向 GitHub 位置的布萊特灣網繭規格。
      • 第 3 行：將 Pods 設置為使用框架而不是靜態庫。
      • 第 5 行：定義 iOS 平台版本。

      • 第 7-9 行：安裝布萊特灣原生播放器 SDK。

        1. 從 iOS 概述文檔的布萊特灣原生 SDK 獲取當前的原生 SDK 版本。
        2. 將值 '6.7.10'取代為目前的 SDK 版本號碼。

        3. 如需有關管理相依性的詳細資訊，請參閱如何使用 Cocoapods 管理相依性文章。

    3. 打開終端會話並導航到您的簡單視頻播放 Xcode 項目文件夾。

    4. 在終端機工作階段中，輸入命令

      pod install

      ，然後按 Return 鍵執行它。

      如果CocoaPods找不到該Pod的兼容版本，或者它不是最新版本，請嘗試在終端應用程序中運行以下命令：

      您應該會在終端中看到一系列訊息，指出 Brightcove 播放程式 SDK 已新增至您的專案。

    • 如果您定期建立和編輯 Podfile，您可以嘗試使用 Cocoapods 應用程式來完成上述步驟。
    • 如果您不想使用 Cocoapods，您可以下載最新版本的 SDK 並手動將其添加到您的項目中。

    設定應用程式的音訊行為

    音訊工作階段會處理應用程式層級的音訊行為。進一步了解音訊會議課程。

    您可以從數個音訊工作階段類別和設定中選擇，以自訂應用程式的音訊行為。為您的應用程式選擇最佳的音訊工作階段類別。

    對於此範例，我們將使用播放類別。即使螢幕鎖定且「鈴聲/靜音」開關設定為靜音，也會播放音訊。由於我們正在播放視頻，我們將使用短片播放模式。為了保持簡單，我們將把這段代碼在應用程序委託。

    1. 在您的項目中，打開應用程序委託文件（ AppDelegate.swift ）。

    2. 在didFinishLaunchingWithOptions函數中，加入程式碼以設定音訊工作階段類別。確保您導入AVFoundation框架。

      • 我們需要下面的代碼，以確保當我們期望音頻播放。舉例來說，如果沒有設定此代碼，靜音開關開啟時，我們就不會聽到視訊。

//  Simple-Video-Playback

import UIKit
import AVFoundation

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {

  var window: UIWindow?

  func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool {
  	// Override point for customization after application launch.

    var categoryError :NSError?
  	var success: Bool
  	do {
  		try AVAudioSession.sharedInstance().setCategory(.playback, mode: .moviePlayback, options: .duckOthers)
  		success = true
  	} catch let error as NSError {
  		categoryError = error
  		success = false
  	}

  	if !success {
  		print("AppDelegate Debug - Error setting AVAudioSession category.  Because of this, there may be no sound. \(categoryError!)")
  	}

  	return true
  }
