當前的版本

版本 6.16.3

2021 年 7 月 1 日

SDK 變化：

  • 添加了對設置標籤屬性的支持Brightcove 字幕格式支持將選定的字幕和音軌信息發送到 Cast 接收器的類
  • 更新了視頻實用程序添加實用方法以從 Video 對像中刪除邊車字幕源的類
  • 修復了一個問題video_duration URL 參數未包含在向 Brightcove 的 Analytics 後端發送報告的請求中

轉換插件的變化：

  • 添加了對為 Cast 接收器應用設置自定義播放器的 URL 的支持。有關更多信息，請參閱 Android SDK 支持文檔

  • 添加了對將選定的字幕和音軌信息發送到 Cast 接收器的支持

    一個已知的非回歸問題將在即將發布的版本中得到解決，如果在播放過程中關閉 Cast 會話，則不會自動在發送器上顯示清單中的字幕選擇。

    此問題的解決方法是，當會話在 Sender 設備上重新打開時，可以重新選擇並顯示在打開 Cast 會話之前所做的選擇。

離線播放更改：

  • 添加了對設置屬性的支持摘要離線目錄班級。可以通過兩種方式設置脫機目錄屬性：

    • 作為一個地圖<字符串，字符串>鍵/值對使用setProperties(Map<String, String>)方法
    • 通過getProperties().put(key, value)方法
  • 添加了對創建自定義的支持通知渠道在裡面下載管理器班級。有關更多信息，請參閱 Android SDK 支持文檔

以前的版本

版本 6.16.2

2021 年 5 月 10 日

SDK 變化：

  • 修復了文本跟踪種類的問題字幕未被解析或顯示
  • 修復了文本軌道具有位置屬性與附加左行或者線右屬性未被解析或顯示

ExoPlayer 2 變化：

  • 修復了可以使用不正確的索引來獲取當前選擇的音頻語言的問題

Omniture 插件更改：

  • 修復了玩家被摧毀後心跳會繼續的問題

SSAI 插件更改：

  • 添加了對嵌套 VAST 和 VMAP 擴展元素的解析支持

    擴展元素，無論嵌套深度如何，都不會由 SSAI 插件執行，因此此支持僅確保解析具有任何深度的嵌套擴展元素的 VAST 和 VMAP 不會失敗。

  • 添加了對 VAST 的解析支持廣告驗證元素

    廣告驗證元素不由 SSAI 插件執行，因此此支持僅確保解析 VAST 廣告驗證元素不會失敗。

    此外，廣告驗證元素是 VAST 4.0 功能，Brightcove Android SDK 尚不完全支持 VAST 4.x。在適用於 Android 的 Brightcove Native SDK 中，這種解析支持通常不應被視為對 VAST 4.x 的支持。

版本 6.16.1

2021年1月12日

SDK 變化：

  • 改進了SSAI VOD視頻的視頻參與度報告。
  • 這DashWidevineAuthorizer類已棄用。請使用Widevine 授權器類代替。

IMA插件更改：

  • Google IMA SDK 已更新至 3.21.1 版。
  • 改進了對可跳過廣告的支持，並解決了以下問題：可跳過廣告可能無法完全退出，並且在恢復內容播放後可能會聽到廣告音頻。
  • 修復了設置廣告的mime類型的問題，該問題導致插件的Builder無法設置用戶提供的mime類型。
  • 添加了用於以下目的的插件構建器便捷方法：
    • 在廣告渲染設置對象 - 見Builder.setMimeTypes(列表） ： 空白
    • 在廣告渲染設置對象 - 見Builder.setLoadVideoTimeout(int) : 無效
    • 設置調試模式ImaSdk 設置對象 - 見Builder.setDebugMode(boolean) : 無效
  • 請注意：

    這些方便的方法是成員方法廣告渲染設置和ImaSdk 設置 Google IMA SDK 對象。當直接通過 Builder 調用時，這些方法將配置廣告渲染設置和ImaSdk 設置在 Builder 中默認創建的對象。

    如果您決定創建自定義廣告渲染設置和ImaSdk 設置用於 IMA 插件的 Google IMA SDK 對象，您可以調用這些對象' 直接方法：

    • AdsRenderingSettings.setMimeTypes（列表） ： 空白
    • AdsRenderingSettings.setLoadVideoTimeout(int) : void
    • ImaSdkSettings.setDebugMode(boolean) : 無效

SSAI 插件更改：

  • 添加了對作為遠程視頻對象創建的實時SSAI流的支持。

版本 6.11.0

2020年2月7日

SDK 變化：

  • 增加了對Brightcove播放權限管理服務的支持。

    請注意，此新功能的可用性有限。

  • 突破性變革：Playlist 對象現在是不可變的。

  • 播放列表對象現在可以包含具有狀態的混合視頻Status.OK和Status.ERROR

  • Playlist.getErrorVideos()：這種新方法將只返回狀態為Status.ERROR

  • Playlist.getAllVideosAndErrors()：這種新方法將返回具有兩種狀態的所有視頻，Status.OK和Status.ERROR , 遵循與從回放權限管理服務響應返回的相同順序

  • 筆記：Playlist.getVideos()和Playlist.getCount()行為與以前的版本相同，也就是說，他們只考慮具有狀態的視頻Status.OK

  • 添加ErrorListener.onError(List)打回來。

  • 不推薦使用ErrorListener.onError(String)打回來。

  • Video 對象現在有一個狀態代碼，可以是Status.OK或者Status.ERROR .您可以通過以下方式檢索狀態Video.getStatus():Status .

  • Status.OK：視頻具有所有預期的屬性，但沒有Video.Fields.ERRORS在裡面。

  • Status.ERROR：視頻有Video.Fields.ERRORS在其性質的形式List<CatalogError> .
